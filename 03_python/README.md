# Python Documentation

This folder contains the Python phase documentation for the **Olist E-Commerce End-to-End Business Intelligence Case Study**.

## Objective

The Python phase focuses on data inspection, quality assessment, preprocessing, feature engineering, exploratory data analysis (EDA), and integration with PostgreSQL to prepare a reliable dataset for business intelligence and dashboard development.

## Documentation Structure

### 01. Python Workflow – Dataset Audit

* Initial dataset inspection
* Data loading workflow
* Data types and structure validation
* Missing value assessment

### 02. Referential Integrity Validation

* Primary key validation
* Foreign key validation
* Relationship verification
* Data consistency checks

### 03. Data Cleaning & Preprocessing

* Missing value handling
* Duplicate detection
* Data type conversions
* Data quality improvements

### 04. Feature Engineering

* Creation of derived features
* Business metrics preparation
* Analytical feature generation

### 05. Python Exploratory Data Analysis (EDA)

* Descriptive statistics
* Data distribution analysis
* Business-oriented exploratory insights

### 06. Python & PostgreSQL Integration

* Exporting cleaned datasets
* Database loading workflow
* Validation before SQL analysis

## Technologies Used

* Python
* Pandas
* NumPy
* PostgreSQL
* Jupyter Notebook
* VS Code

## Outcome

The outputs from this phase provide a clean, validated, and analysis-ready dataset that serves as the foundation for SQL analysis and Power BI dashboard development.
